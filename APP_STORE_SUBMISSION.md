# App Store Submission Guide for Clean & Sober

## Prerequisites

Before submitting to app stores, ensure you have:

1. **Apple Developer Account** ($99/year) - [developer.apple.com](https://developer.apple.com)
2. **Google Play Developer Account** ($25 one-time) - [play.google.com/console](https://play.google.com/console)
3. **macOS with Xcode** (for iOS builds)
4. **Android Studio** (for Android builds)

## Setup Steps

### 1. Clone Your Repository

```bash
git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Add Native Platforms

```bash
npx cap add ios
npx cap add android
```

### 4. Build the Web App

```bash
npm run build
```

### 5. Sync with Native Projects

```bash
npx cap sync
```

### 6. Update for Production

Before submitting, update `capacitor.config.ts`:

```typescript
const config: CapacitorConfig = {
  // Remove or comment out the server config for production
  // server: {
  //   url: '...',
  //   cleartext: true
  // },
  webDir: 'dist',
  // ... rest of config
};
```

Then rebuild and sync:

```bash
npm run build
npx cap sync
```

## iOS Submission

### Open in Xcode

```bash
npx cap open ios
```

### Configure in Xcode

1. Select your project in the navigator
2. Set your **Team** under Signing & Capabilities
3. Update **Bundle Identifier** if needed
4. Set **Version** and **Build** numbers
5. Add required **App Icons** (see Assets.xcassets)
6. Configure **Launch Screen**

### App Store Connect

1. Create your app in [App Store Connect](https://appstoreconnect.apple.com)
2. Fill in all metadata (name, description, keywords, etc.)
3. Upload screenshots for all required device sizes
4. Set pricing and availability
5. Complete privacy policy and age rating

### Build & Submit

1. In Xcode: Product → Archive
2. Distribute App → App Store Connect
3. Wait for processing, then submit for review

## Android Submission

### Open in Android Studio

```bash
npx cap open android
```

### Generate Signed APK/Bundle

1. Build → Generate Signed Bundle/APK
2. Create a new keystore (save this securely!)
3. Select "Android App Bundle" for Play Store
4. Choose "release" build variant
5. Build

### Google Play Console

1. Create your app in [Google Play Console](https://play.google.com/console)
2. Fill in store listing (title, description, graphics)
3. Upload screenshots for phone and tablet
4. Set content rating
5. Configure pricing and distribution
6. Upload your signed AAB file
7. Submit for review

## Privacy Policy

You need a privacy policy URL. Create one that covers:
- Data collection practices
- How data is used
- Third-party services (Supabase, Stripe)
- User rights (GDPR, CCPA compliance)
- Contact information

## Support URL

Provide a support email or webpage for user inquiries.

## Testing Checklist

Before submission, test:

- [ ] App launches without crashes
- [ ] Sign up / Sign in works
- [ ] All main features functional
- [ ] In-app purchases work (if applicable)
- [ ] Push notifications work
- [ ] Offline behavior is acceptable
- [ ] No placeholder content
- [ ] All links work
- [ ] Privacy policy accessible

## Common Rejection Reasons

1. **Incomplete information** - Fill all required fields
2. **Bugs or crashes** - Test thoroughly
3. **Placeholder content** - Remove all Lorem ipsum
4. **Privacy issues** - Proper consent for data collection
5. **Misleading metadata** - Accurate descriptions only
6. **Login issues** - Provide demo account if needed
